# BeerPub — система управления пивным баром

Веб-приложение для учёта товаров, заказов, накладных, кассовых смен и отчётов. Поддерживает составные товары (комплекты) с автоматическим списанием компонентов по FIFO.

## Быстрый старт

```bash
docker-compose up -d
```

После запуска:

| Сервис    | URL                          |
|-----------|------------------------------|
| Frontend  | http://localhost:8080        |
| API docs  | http://localhost:8000/docs   |
| PostgreSQL | localhost:5433              |

### Учётные данные приложения

| Логин      | Пароль   | Роль       |
|------------|----------|------------|
| admin      | beer123  | Администратор |
| bartender  | beer123  | Бармен     |

Подключение к БД: `localhost:5433`, пользователь/пароль/БД — из файла `.env`.

## Стек

**Backend:** FastAPI, SQLAlchemy 2, PostgreSQL 15, JWT  
**Frontend:** React 18, Ant Design 5, Vite, Recharts  
**Инфра:** Docker Compose, Nginx

## Основные возможности

- **Товары** — простые позиции и комплекты (kit) из нескольких компонентов
- **Партии (FIFO)** — приход через накладные, списание по принципу «первым пришёл — первым ушёл»
- **Заказы** — при продаже комплекта автоматически разворачиваются и списываются все компоненты
- **Чеки** — основная позиция комплекта + компоненты с `show_in_receipt=true`
- **Касса** — смены; нельзя создать заказ без открытой смены и закрыть смену с открытыми заказами
- **Отчёты** — продажи, маржа, топ товаров, график выручки

## Комплекты

Примеры из seed-данных:

| Комплект                  | Компоненты                                      |
|---------------------------|-------------------------------------------------|
| Жигулевское 0.5 зал       | 0.5 л пива + бокал (оба в чеке)                 |
| Жигулевское 1.5 навынос   | 1.5 л пива (в чеке) + бутылка и крышка (скрыты) |
| Чешское 0.5 зал           | 0.5 л пива + бокал                              |
| IPA 0.3 зал               | 0.3 л IPA                                       |

Цена комплекта может быть **ручной** (`retail_price`) или **автоматической** (сумма цен компонентов).

## API

Полная документация доступна в Swagger UI: http://localhost:8000/docs

Основные группы эндпоинтов:

- `/api/auth` — авторизация (JWT)
- `/api/products` — товары и комплекты
- `/api/invoices` — приходные накладные
- `/api/orders` — заказы (создание, оплата, отмена)
- `/api/receipt` — данные для печати чека
- `/api/cash` — кассовые смены
- `/api/reports` — отчёты и дашборд
- `/api/batches` — партии и движения

## Локальная разработка (без Docker)

### Backend

```bash
cd backend
pip install -r requirements.txt
# PostgreSQL должен быть доступен, DATABASE_URL в .env
uvicorn app.main:app --reload --port 8000
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Dev-сервер Vite проксирует `/api` на `http://localhost:8000`.

## Остановка

```bash
docker-compose down
```

Для удаления данных БД:

```bash
docker-compose down -v
```
